# Query Skill

## 目标
用最少检索成本给出可追溯答案：优先 `wiki/`，必要时补查 `sources/`。

## 执行流程

1. 先查 wiki。
   - 读取 `wiki/index.md` 定位页面
   - 读取相关 `wiki/concepts/*.md`
   - 必要时看 `wiki/log.md` 获取最近更新上下文

2. 若 wiki 信息不足，再回溯 sources。
   - 到 `sources/<org>-<repo>/` 定位原文
   - 只补充回答所必需的信息

3. 组织答案。
   - 先给结论，再给关键依据
   - 多概念问题用小标题或表格对比
   - 末尾给“建议下一步阅读页面”

4. 可选回写（仅在用户同意时）。
   - 如果查询过程中形成了高价值新总结，可回写到对应 wiki 页面

## 输出约束

- 回答必须可追溯：引用具体文件路径与行号。
- 不确定就明确说明“当前 wiki 未覆盖”，并指出缺口页。
- 避免泛泛而谈，聚焦用户问题。
